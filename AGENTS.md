# 🧾 공동 자산 목표 관리 웹 서비스  
**개발 요구사항 명세서 (Development Requirements Specification)**  
**버전:** v2.1
**작성일:** 2025-10-11  
**작성자:** 김세호
---

## 1. 프로젝트 개요

### 1.1 목적
두 사용자가 공동으로 자산 목표를 설정하고, 각자의 지출/저축 내역을 공유하며,  
목표 달성 진행 상황을 함께 관리할 수 있는 웹 서비스를 개발한다.  

서비스의 핵심은 **투명한 자산 관리**, **동기 부여**, **실시간 협업**이다.

### 1.2 목표
- 간결하고 직관적인 UI로 자산 현황을 쉽게 파악  
- 유지보수성과 확장성을 고려한 시스템 구조  
- Firebase 등 외부 유료 서비스 의존 최소화로 **비용 절감**

## 2. 시스템 개요

| 구분 | 내용 |
|------|------|

## 3. 기능 요구사항

### 3.1 사용자 관리

| 기능 | 설명 |
|------|------|
| 회원가입 | 사용자 ID, 비밀번호 입력 후 등록. 비밀번호는 **bcrypt** 해시 저장 |
| 로그인 | ID/PW 인증 후 JWT 토큰 발급 |
| 로그아웃 | 클라이언트 단 토큰 제거 방식 |
| 비밀번호 찾기 | 이메일 기반 임시 비밀번호 발급 (SMTP 연동) |
| 공동 사용자 초대 | 초대 코드 또는 이메일 기반 파트너 연결 기능 |

---

### 3.2 자산 목표 관리

| 기능 | 설명 |
|------|------|
| 목표 생성 | 목표명, 목표금액, 기간, 설명 입력 |
| 목표 공유 | 두 사용자가 동일 목표를 공유 가능 (소유자 / 참여자 권한 구분) |
| 목표 진행률 표시 | 누적 저축 금액 대비 달성률을 게이지 또는 차트로 시각화 |
| 목표 수정 및 삭제 | 권한이 있는 사용자만 수행 가능 |

---

### 3.3 자산 내역 관리

| 기능 | 설명 |
|------|------|
| 입금/지출 등록 | 금액, 날짜, 카테고리, 메모 등록 |
| 공동 분담 관리 | 두 사람의 분담 비율 및 총액 합산 계산 |
| 내역 조회 | 월별, 카테고리별 필터 기능 제공 |
| 시각화 | 월별 지출 그래프 및 저축 진행률 차트 표시 |

---

### 3.4 리포트 및 알림

| 기능 | 설명 |
|------|------|
| 월간 리포트 | 월별 수입/지출 요약 리포트 (화면 출력 중심, PDF 내보내기 향후 지원) |
| 목표 달성 알림 | 달성률 50%, 100% 도달 시 UI 상 알림 |
| 예측 기능 | 목표 대비 남은 금액 및 예상 달성 시점 표시 (간단 계산 기반) |

---

### 3.5 기타 공통 기능

| 기능 | 설명 |
|------|------|
| 반응형 UI | 데스크탑/모바일 대응 |

## 4. 비기능 요구사항

| 구분 | 요구사항 |
|------|-----------|

| **테스트** | Jest (React), Pytest (FastAPI) 기반 단위 테스트 수행 |
| **로그 관리** | GCP Cloud Logging 및 Error Reporting 사용 |
| **데이터 백업** | Cloud SQL 자동 백업 주기 설정 (1일 1회) |

## 5. 기술 스택 상세

| 구분 | 기술 | 비고 |
|------|------|------|
| **Frontend** | React.js (v18+), Material UI, Axios, React Router | SPA 구조, 컴포넌트 재사용 |
| **Backend** | FastAPI (Python 3.11+), Uvicorn, SQLAlchemy | 비동기 처리 및 Swagger 문서 자동화 |
| **Database** | MySQL (Cloud SQL or VM 설치형) | ORM(SQLAlchemy) 기반 접근 |
| **Auth** | JWT + bcrypt | 자체 인증 구조 |
| **Infra** | GCP Cloud Run (API), Cloud SQL (DB), Cloud Storage (정적 자원) | 자동 확장 및 관리형 배포 |
| **CI/CD** | GitHub Actions + GCP Cloud Build | main/dev/feature 브랜치 전략 |

## 7. 화면 설계 개요

| 화면명 | 주요 기능 |

| 로그인 화면 | ID/PW 입력, 회원가입 링크 |

## 8. 데이터베이스 개요 (요약형 ERD)

| 테이블 | 주요 컬럼 | 설명 |
|---------|------------|------|
| **user** | id, email, password_hash, name, created_at | 사용자 계정 정보 |
| **auth_token** | user_id, token, expires_at | JWT 토큰 관리 (선택 사항) |
